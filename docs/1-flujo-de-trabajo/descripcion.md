# Propósito de este Capítulo

Este capítulo tiene como finalidad describir el flujo de trabajo que se debe seguir para construir, validar y desplegar un mapa web en la Sección de Georreferenciación.

A diferencia de otros capítulos de este documento —donde se abordan aspectos como el diseño de la interfaz, la arquitectura de desarrollo, el stack tecnológico o el consumo de servicios—, este capítulo se centra en explicar el proceso completo de desarrollo, desde el surgimiento de la necesidad o idea inicial, pasando por el desarrollo y la validación de funcionalidades, hasta la publicación final del mapa web.

El objetivo principal de este capítulo es entregar una guía práctica y ordenada del “cómo” se debe desarrollar un mapa web, estableciendo un flujo común para todo el equipo, que permita mantener el orden, la trazabilidad de los cambios y el trabajo colaborativo, especialmente en el uso de herramientas de control de versiones como Git y GitHub.

Este capítulo no profundiza en detalles técnicos específicos, salvo aquellos estrictamente necesarios para comprender el flujo de trabajo propuesto. Se invita a los lectores de este manual a complementar la información aquí presentada con los capítulos siguientes y con la profundización técnica que cada uno requiera para su constante mejora en el desarrollo de mapas web.
